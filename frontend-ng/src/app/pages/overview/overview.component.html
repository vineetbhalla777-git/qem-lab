How the pipeline works

Every experiment on this site runs the same four-stage pipeline against a real Qiskit + Mitiq backend.

Stage 1

Simulate the hardware

A tunable noise profile generates depolarizing error, T1/T2 decoherence, and asymmetric readout error at a chosen strength.

Stage 2

Run the benchmark

A standard circuit — Bell state, GHZ, QFT, VQE, or QAOA — executes on the noisy simulator via Qiskit Aer.

Stage 3

Mitigate

One of six techniques (ZNE, MEM, PEC, CDR, DDD, or Virtual Distillation) processes the noisy result.
